# GRICH Nginx 配置手动步骤

## 问题说明
应用在 3005 端口运行正常，但 Nginx 代理配置有问题。需要手动创建正确的配置文件。

## 操作步骤

### 1. 在 SSH 终端执行以下命令创建 Nginx 配置

```bash
sudo tee /etc/nginx/sites-available/jaxfamlaw.conf > /dev/null << 'EOF'
server {
    listen 80;
    server_name jaxfamlaw.com www.jaxfamlaw.com;
    location / {
        proxy_pass http://localhost:3005;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
    }
}
EOF
```

### 2. 创建符号链接

```bash
sudo ln -sf /etc/nginx/sites-available/jaxfamlaw.conf /etc/nginx/sites-enabled/
```

### 3. 测试 Nginx 配置

```bash
sudo nginx -t
```

应该看到：`configuration file /etc/nginx/nginx.conf test is successful`

### 4. 重新加载 Nginx

```bash
sudo systemctl reload nginx
```

### 5. 测试访问

```bash
curl -I http://localhost/compliance/Nike
```

应该看到 `HTTP/1.1 200 OK`

## 完成后
在 Cloudflare 开启橙色云，然后访问：
http://jaxfamlaw.com/compliance/Nike
